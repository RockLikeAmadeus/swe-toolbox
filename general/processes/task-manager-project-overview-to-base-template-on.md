# Task Manager (Actual Name TBD, AnteTodo?)

## What am I trying to solve? What do I need this tool to accomplish?

- I need to have a database of all the things I need to do. If it's not in this tool, or in my calendar for a specific time, I can't trust that it will get done.
- I need to be able to track small one of tasks that I can do in one sitting, as well as large multi-step nested projects that will take months or years to complete, and everything in between.
- I need a way to filter this massive list down to the most important and urgent things, so that I can hold it in my mind (or better, display it in this tool) and decide what to do at any given time.
- For projects that need to be completed at some point, but can't be done all at once, I need a way to make sure I'm making enough progress on it now so that I'm on track to complete it when I expect to.
- I need a way to prevent having to look at every task every day, but still be confident that nothing will fall through the cracks or get forgotten about.
- I need the system to be flexible and provide ways to adjust to change at larger time horizons (i.e. if I have something planned to finish this year, but it becomes no longer a priority, I need to be able to realize this and update my plan more frequently than yearly)

## What are some ways I can solve these problems and accomplish these goals with this system?

- Have fixed (not rolling) date horizons (today, this week, this month, this quarter, etc.) that are _configurable_ (so that this works for work as well, i.e. today, this week, this sprint, etc.) and projects, subprojects, or tasks that are configured within one of these date horizons (or an unscheduled category)
  - Have views for each date horizon that clearly show tasks that were scheduled for this date horizon, completed and completed during this date horizon, and how far through this date horizon I am in time.
  - Walkthroughs for reviews for each date horizon that are configurable to some extent, but flexible relative to any date horizon.
- A way to mark projects and subprojects as "active", which essentially means that projects or subproject is scheduled within the current date horizon, with a view that displays active projects in order of time horizon. But how woudl this work for subprojects?
